    键盘事件详解

    Paradox 5/29/2021 javascriptvue

    # 一、按键事件基础

    1. keydown: 键盘按下事件(按住键盘不松，会多次触发)
    2. keypress: 接着 keydown 事件触发(只有按下字符键时触发，比如数字，字母，符号， 也会多次触发)
    3. keyup：键盘释放事件，键盘释放时触发

    # 二、事件注册

    键盘事件中可以在函数中传入参数， 或者在函数中直接调用 event

    <script>
    window.onload = () => {
      document.onkeydown = () => {
        console.log(event)
      }
      document.addEventListener('keydown', () => {
        console.log(event)
      })
    }
  </script>

    # 三、触发过程

    1. 用户按下字符键
    • 首先会触发keydown事件
    • 然后紧接着触发keypress事件
    • 最后触发keyup事件
    • 如果用户按下了一个字符键不放，就会重复触发keydown和keypress事件，直到用户松开该键为止
    1. 用户按下非字符键
    • 首先会触发keydown事件
    • 然后就触发key事件
    • 如果用户按下了一个非字符键不放，就会重复触发 keydown 事件，直到用户松开该键为止

    ​

    # 四、获取按键值

    1. KeyboardEvent.charCode (opens new window) 和 KeyBoardEvent.keyCode (opens new window)已从 web 标准中删除
    2. 认为当前环境下，浏览器厂商应该会一直保持支持，还是继续可以使用 😂
    3. 推荐使用 KeyboardEvent.key (opens new window) 属性确定键盘事件， KeyboardEvent.key 键值表 (opens new window)

    ​

    # 五、注意事项

    1. keypress 无法响应系统功能键如(delete, backspace)
    2. keypress 在中文输入过程中是无法触发的
    3. keyup中无法阻止浏览器默认事件， 若需阻止浏览器默认事件，应该在keydonw或keypress中阻止
    4. keyup，keydown 可以监听ctrl、alt、shift键，而当按下系统功能键时再按字符键，无法触发keypress事件

    # 六、组合按键

    1. 与 ctrl、alt、shift键组合， 可以直接再 keydown 事件中判断ctrlKey、altKey、shiftKey的值
    document.addEventListener("keydown", () => {
  if (event.ctrlKey && event.key === "c") {
    console.log("key down ctrl + c");
  }
});

    1. 多个字母键组合，手动实现，再keydown事件将键位存入， keyup事件时释放
    let key = {};
document.addEventListener("keydown", () => {
  key[event.key] = 1;
  if (key["b"] && key["c"]) {
    console.log("key down b + c");
  }
});
document.addEventListener("keyup", () => {
  delete key[event.key];
});

    # 七、VUE 中使用键盘事件

    1. 在 mountd 中注册
    mounted () {
    window.addEventListener('keyup',this.handleKeyup)
    window.addEventListener('scroll',this.handleScroll)
},
    1

    1. 在元素中使用, 按键修饰符参考 vue 按键修饰符 (opens new window)
    <input @keyup.page-down="onPageDown">
    1
    1. 在对于已经封装的组件，需要加上native修饰符,native 修饰符可以让父组件接收到原生事件
    <el-input @keyup.enter.native="handleEnterInput"></el-input>
